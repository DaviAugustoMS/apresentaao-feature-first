Fluxo de Dados

Screen ViewModel API

screens/

VIEW

Containers finos ("Dumb Components"). Apenas renderizam dados e disparam eventos do ViewModel.

viewModels/

VIEWMODEL

Orquestração da lógica. Recebe input do usuário, valida e chama os hooks de dados.

hooks/

MODEL

Server State via React Query. Gerencia cache, loading states, erros e retries.

api/

MODEL

Comunicação HTTP pura (Axios). Definição de endpoints e tipagem de retorno (DTOs).

components/

Componentes visuais específicos desta feature (ex: CartItem, PaymentMethodCard).

index.ts

Public API controlada. Exporta apenas o necessário (Screens principais, hooks públicos).

Benefício Real

Alterar a regra de cálculo do carrinho não quebra a tela de pagamento. A lógica vive no ViewModel/Hook, não na UI.
