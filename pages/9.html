Feature First 4. Nova Estrutura

Arquitetura Orientada a Domínio

starbem-v2
src/features/
├── auth/ // Login, Registro, MFA
├── buyProduct/ // Sacola, Wallet, Pix
├── feelings/ // Diário, Clean Arch
└── schedule/ // Agendamento (WIP)
// Nota: Cada pasta contém tudo // que a feature precisa para existir.

Benefícios Estruturais

Encapsulamento Real

Uma feature pode ser removida deletando apenas uma pasta. Sem "caça ao tesouro".

Colocation (Localidade)

Tudo que muda junto, fica junto. O código relacionado está a 1 clique de distância.

Regra de Ouro

"Se for específico da feature → fica dentro da feature."

"Se for compartilhado → src/shared/"
