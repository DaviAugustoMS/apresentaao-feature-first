Legado 2. Estrutura Antiga

O Monólito Técnico

starbem-legacy
src/
├── hooks/ // Lógica de negócio + API
├── screens/ // UI + Lógica de Apresentação
├── store/ // Estado Global Compartilhado
├── services/ // Chamadas HTTP soltas
├── components/ // Componentes (UI Kit?)
└── pages/ // Rotas e Navegação

Diagnóstico

Responsabilidades Espalhadas

Para criar uma feature, você precisa tocar em hooks, store e screens simultaneamente.

Acoplamento Invisível

Hooks e Stores globais criam dependências implícitas difíceis de rastrear e refatorar.

Risco de Regressão

Alterar um hook "genérico" pode quebrar telas não relacionadas silenciosamente.

Organização por tipo técnico = Gaveta de cabos bagunçada
